# InfoVis

## LECTURE 7

Dynamic visualization 	
As we seek information we filter and go back and forth between and overview view and specific detail. This is the definition of a dynamic system. A propos to schneiderman's motto: overview first, zoom and filter, then details on demand.

Examples
- Treemaps are an example, e.g. www.newsmap.jp
- LexisNexis interactive map view
- Fisheye views, e.g. table lens that blows up specific data row, fisheye menu, apple dock
- Semantic network with nodes and and links
- Animation
Characteristics
- Overview and detail displays show data at more than one level
- Linking and brushing using multiple techniques to overcome the shortcomings of an individual display. Brushing refers to make a selection and comparing that same data across multiple displays, I think.
